<task type="interactive">
  <name>Task 3: Deploy to Vercel</name>
  <files>
  </files>
  <action>
Deploy the portfolio to Vercel's free Hobby tier.

**Step 1: Check if Vercel CLI is installed**

```bash
npx vercel --version 2>/dev/null || echo "Vercel CLI not found"
```

**Step 2: Deploy (requires user interaction)**

Run the deploy command. The user will need to:
1. Authenticate with their Vercel account (or create one — free)
2. Confirm project settings when prompted

```bash
cd D:/goat && npx vercel --yes
```

The `--yes` flag auto-confirms default settings. Vercel will:
- Detect Next.js project
- Build and deploy automatically
- Assign a URL like `kazamiyuuji-<team>.vercel.app`

**Step 3: Capture the deployment URL**

The CLI output will contain the deployment URL. Record it for verification.

**If deploy fails:** Check that the user has a Vercel account and is authenticated. Common issues:
- Not logged in: `npx vercel login`
- No team: Accept default team during deploy
- Build errors: Run `npm run build` locally first to verify
  </action>
  <checkpoint>
User must authenticate with Vercel during deploy. If they don't have an account, they need to sign up first at vercel.com.
