### 4. Root Layout (`app/layout.tsx`)

**Source:** `RESEARCH:TECH-STACK.md` (project structure), `CONTEXT.md` lines 11, 33

**Font setup pattern (Outfit from Google Fonts via `next/font`):**
```tsx
import type { Metadata } from "next";
import { Outfit } from "next/font/google";
import "../styles/globals.css";

const outfit = Outfit({
  subsets: ["latin"],
  variable: "--font-outfit",
});

export const metadata: Metadata = {
  title: "Kazamiyuuji - Portfolio",
  description: "Personal portfolio with Frutiger cyber aesthetic",
};

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en" className={outfit.variable}>
      <body className="font-sans antialiased">
        {children}
      </body>
    </html>
  );
}
```

**Key decisions:**
- Font loaded via `next/font/google` (self-hosted, no CLS, automatic `font-display: swap`)
- CSS variable `--font-outfit` for Tailwind integration
- Server Component by default (no `"use client"` directive)

```tsx
interface WaveProps {

export function Wave({ className = "", flip = false }: WaveProps) {
  return (
    <div className={`w-full overflow-hidden ${flip ? "rotate-180" : ""} ${className}`}>

          className="text-frost"

}
```

## Shared Patterns

### Tailwind v4 CSS-First Configuration
**Apply to:** `styles/globals.css`
**Source:** `RESEARCH:TECH-STACK.md` lines 65-73

Key differences from v3:
- NO `tailwind.config.js` file
- Configuration via `@theme` blocks in CSS
- Custom colors defined as `--color-*` CSS variables
- Custom fonts defined as `--font-*` CSS variables
- Breakpoints defined as `--breakpoint-*` CSS variables

### Font Loading via `next/font`
**Apply to:** `app/layout.tsx`
**Source:** `RESEARCH:TECH-STACK.md` (pitfall #10)

```tsx
import { Outfit } from "next/font/google";
const outfit = Outfit({ subsets: ["latin"], variable: "--font-outfit" });
```

This self-hosts the font, applies `font-display: swap` automatically, and eliminates CLS.
