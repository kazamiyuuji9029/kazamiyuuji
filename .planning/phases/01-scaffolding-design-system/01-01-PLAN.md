**Step 1: Initialize Next.js 16**

```bash
npx create-next-app@latest . --typescript --eslint --app --tailwind --src-dir=false --import-alias="@/*" --use-npm
```

Run from project root (D:/goat). When prompted, select:
- TypeScript: Yes
- ESLint: Yes
- Tailwind CSS: Yes
- src/ directory: No
- App Router: Yes (default)
- import alias: @/*

This creates the base scaffolding with App Router and Tailwind v4.

**Step 2: Verify Tailwind v4 setup**

After create-next-app completes, verify the project has Tailwind v4 (not v3):

```bash
cat package.json | grep tailwindcss
```

Expected: `"tailwindcss": "^4.x"` (v4.x, NOT v3.x).

If v3 was installed (older create-next-app cache), remove and reinstall:

```bash
npm uninstall tailwindcss @tailwindcss/postcss postcss
npm install tailwindcss @tailwindcss/postcss postcss
```

**Step 3: Configure postcss.config.mjs**

Replace the generated postcss.config.mjs with:

```js
/** @type {import('postcss-load-config').Config} */
const config = {
  plugins: {
    "@tailwindcss/postcss": {},
  },
};

export default config;
```

IMPORTANT per RESEARCH: Use `@tailwindcss/postcss` NOT `require('tailwindcss')`. The old PostCSS plugin syntax causes build errors in v4.

**Step 4: Configure next.config.ts for static export**

import type { NextConfig } from "next";

const nextConfig: NextConfig = {

export default nextConfig;
```

IMPORTANT per RESEARCH: `output: 'export'` requires `images: { unoptimized: true }` because Image Optimization needs a server.

import { ReactNode } from "react";

interface GlassPanelProps {
  children: ReactNode;
  className?: string;
  variant?: "default" | "hero" | "contact";
}

export function GlassPanel({
  children,
  className = "",
  variant = "default",
}: GlassPanelProps) {
  const variantStyles = {
    default: "glass-panel p-8",
    hero: "glass-panel p-12 md:p-16",
    contact: "glass-panel p-8 md:p-12",
  };

  return (
    <div className={`${variantStyles[variant]} ${className}`}>
      {children}
    </div>
  );
}
```
